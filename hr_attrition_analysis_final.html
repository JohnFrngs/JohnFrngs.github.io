IBM HR Analytics Employee Attrition

The IBM HR Analytics Employee Attrition is a fictional dataset that helps us explore key factors that may influence employee attrition, aiming to uncover patterns that help the Human Resources team make data-driven decisions to improve employee retention.

Project Objectives

By leveraging Python for data handling and Tableau for visual storytelling, I aim to answer questions such as:

  • What roles or departments experience higher attrition?
  • Are there common patterns among employees who leave the company?
  • How do income, satisfaction, and overtime relate to attrition risk?

This analysis serves as a demonstration of applied data science techniques across Python (numpy, pandas, matplotlib, statsmodels, seaborn, scikitlearn) and interactive data visualization platform, Tableau.

Categorical Variable Frequency Summary

Before diving into statistical testing, it's essential to understand the distribution of each categorical variable in the dataset. The table below summarizes:

  • Variable Name: The categorical feature being described
  • Category: Each distinct value within the variable
  • F (Frequency): Number of observations in each category
  • RF (Relative Frequency): Proportion of total entries in that category
  • CF (Cumulative Frequency): Running total of observations as we move through categories
  • CRF (Cumulative Relative Frequency): Cumulative proportion of observations
In[14]:
import pandas as pd

# Define categorical variables
categorical_cols = [
    'businesstravel', 'department', 'education',
    'educationfield', 'environmentsatisfaction', 'gender',
    'jobinvolvement', 'joblevel', 'jobrole', 'jobsatisfaction',
    'maritalstatus', 'overtime', 'performancerating',
    'relationshipsatisfaction', 'stockoptionlevel',
    'worklifebalance'
]

# Create frequency summary
freq_summary = []

for col in categorical_cols:
    freq = df[col].value_counts().sort_index()
    total = freq.sum()
    rf = freq / total
    cf = freq.cumsum()
    crf = rf.cumsum()

    for category in freq.index:
        freq_summary.append({
            'Variable': col,
            'Category': category,
            'F': freq[category],
            'RF': round(rf[category], 4),
            'CF': cf[category],
            'CRF': round(crf[category], 4)
        })

# Create and display DataFrame
freq_df = pd.DataFrame(freq_summary)
freq_df

results = []

    # T-test
    t_stat, t_p = ttest_ind(group_yes[var], group_no[var], equal_var=False)
    df_t = len(group_yes[var]) + len(group_no[var]) - 2

    # Mann-Whitney U test
    u_stat, u_p = mannwhitneyu(group_yes[var], group_no[var], alternative='two-sided')

    results.append({

        't-statistic (df)': f"{t_stat:.3f} ({df_t})",
        't-test p-value': round(t_p, 5),
        'Mann-Whitney U': round(u_stat, 3),
        'MWU p-value': round(u_p, 5)
    })

Conclusion

This project set out to explore and predict employee attrition using the IBM HR Analytics dataset. We began by thoroughly cleaning and preparing the data, ensuring that only meaningful, non-redundant features were included. Through an exploratory data analysis, we identified key patterns in the datasuch as a wide spread in monthly income, varied tenure in roles, and notable differences in overtime and job satisfactionthat are associated with employee turnover.

We then applied a series of modeling approaches:

  • Logistic Regression provided a clear, interpretable model where each unit increase in factors like overtime or years since last promotion significantly impacted the odds of attrition.
  • Random Forest captured complex interactions among variables, highlighting the predictive strength of features such as overtime and years in current role, while also revealing the challenge of class imbalance.
  • XGBoost, with proper class imbalance handling, delivered competitive performance and improved recall for the minority attrition class.

Key insights from the models include:

  • Overtime emerged as the most dominant predictor, with employees working overtime having substantially higher odds of leaving.
  • Years in current role and monthly income are also critical, indicating that tenure and compensation play crucial roles in retention.
  • The balance between model performance and interpretability was maintained by selecting models that not only achieved high accuracy but also provided actionable insights for HR decision-making.
